# Handoff Notes (2026-03-18)

## Summary
- Top stats card filters were stuck because they used `detailed_orders`, which is capped at 100 rows.
- For large customers (e.g., `121.1.049`), the "Πρόσφατες εκτελεσμένες" card would max at 100 and not change.

## Fix Implemented
Added uncapped `range_summary` aggregates on the backend and wired the cards to use them.

### Backend
- `site/lib/customer-stats/sqlite-provider.js`
  - Added `rangeSummary` for non-imported and imported paths.
  - Included `range_summary` in response payload.
- `site/lib/customer-stats/shared.js`
  - Preserves `range_summary` through `normalizeStatsPayload`.
- `site/lib/customer-stats/entersoft-provider.js`
  - Passes through `range_summary` if provided by Entersoft.

### Frontend
- `site/public/admin.js`
  - Added `rangeSummaryCache` and `fetchRangeSummary(...)` for `/stats?sales_time_range=<range>`.
  - Cards read from cached `range_summary` when available, fallback to filtered `detailed_orders`.
  - `fetchAllRangeStats` caches the `all` range summary.
  - Cache cleared in `resetStats`.

## Behavior
- Card filters now load correct totals for any range, no 100-row cap.
- Changing a card range triggers background fetch and re-render.
